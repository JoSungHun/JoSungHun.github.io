webhacking.kr 5번문제

처음 문제에 들어가면 login 버튼과 join 버튼이 있다. login 버튼을 누르면 mem/login.php로 이동하지만 join 버튼을 누르면 no() 함수가 실행되어 alert(‘Access_Denied’)가 실행된다.

login.php가 아닌 join.php로 들어갈 수 있을것이라 추측하고 들어가보면 bye라는 alert가 뜨고 소스코드를 보면 스크립트가 있는것을 확인할 수 있다.

l = a, ll = b 이런식으로 되어 있고 밑에는 스크립트 코드가 있다. 읽기 어렵게 해둔 부분을 개발자도구 Console을 이용하여 무슨 값을 나타내는지 볼 수 있다.

보기 좋게 바꿔서 스크립트를 다시 써보면 다음과 같다.

코드를 보면 oldzombie 라는 쿠키값이 없거나 url에 mode=1이 없으면 alert가 뜬다. 그리고 둘다 만족한다면 밑에 else문이 실행되게 된다.

else문에 있는 코드를 보면 join.php로 값을 보내는데 post로 id, pw값을 전송한다. burpsuite를 사용하여 id, pw값을 전송해 본다.

전송하게되면 sign up as test success 라는 글이 보인다.

가입한 test로 가입하니까 admin으로 로그인하라고 한다…

다시 admin으로 가입하면 이미 있는 아이디라고 나온다. 낚시를 당한듯 하다.

1’ or ‘1’=’1 같은 sql 문을 넣어도 문제가 풀리지 않았다. 위에 자바스크립트에서 id값의 maxlength=20인것으로 보아 id길이를 프론트단에서만 확인하는것 같아 가입할때 admin 하고 공백을 매우 많이 넣어 가입을 해본다.

admin/admin으로 가입하면 문제가 풀린다! 가입하는 페이지가 낚시가 아니였다.
